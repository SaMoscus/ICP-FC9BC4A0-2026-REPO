# Portfolio Presentation

## Title

DevOps Internship Portfolio - Moses Odom

## Introduction

This portfolio presents the completed work for the 6-week InternCareerPath
DevOps internship. The work focused on practical automation, Linux operations,
Git usage, Docker containerization, reliability testing, and documentation.

## Project 1 Summary

**Name:** Shell Scripting Automation  
**Problem:** Repetitive system administration tasks need automation.  
**Solution:** Bash scripts were created to automate backups, cleanup, logging,
and monitoring activities.

**Tools Used:**

- Bash
- Linux terminal
- Cron planning
- Git and GitHub

**Result:**

The project produced working scripts with log output and backup artifacts,
showing how repetitive operational tasks can be automated.

## Project 2 Summary

**Name:** Dockerize an Application  
**Problem:** Applications need consistent environments across machines.  
**Solution:** A static web application was packaged into an Nginx Docker
container and managed with Docker Compose.

**Tools Used:**

- Docker
- Docker Compose
- Nginx Alpine
- HTML and CSS

**Result:**

The project produced a containerized application that can be built and run with a
single Compose command.

## Lessons Learned

- Clear folder structure makes DevOps projects easier to maintain.
- Logs and error messages are important for troubleshooting automation.
- Docker improves environment consistency.
- Documentation is part of the deliverable, not an afterthought.

## Closing Statement

The internship produced two completed DevOps projects and a documented portfolio
that demonstrates automation, containerization, testing, and operational
readiness.
